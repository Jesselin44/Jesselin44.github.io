Project Documentation

Overview

This web application is a simple trivia quiz

  • HTML for structure
  • Bootstrap for layout and styling
  • Vanilla JavaScript and fetch() for API calls

The Rest API Used:

The app uses the Open Trivia Database API: https://opentdb.com/api_config.php

Endpoints incorporated:

  • Categories Endpoint
    https://opentdb.com/api_category.php
    Which is used on index.html to load the list of trivia categories into the dropdown itself.
  • Questions Endpoint
    https://opentdb.com/api.php?amount=5&category=9&difficulty=easy
    Which was used on quiz.html to load quiz questions based on the users preferences.

How to Use the Application

  1. Wait for the Category dropdown to load.
  2. Select a category (or leave as is).
  3. Select a difficulty level (or leave as is).
  4. Choose the number of questions (1-20), 20 being the max amount of questions per round.
  5. Click Begin Quiz.
  6. Answer each question by clicking on an answer button.
  7. At the end, view your score and choose to play again with the same settings or go back home.

Functionality Summary

  • fetch() was used to retrieve data from the API.
  • Dynamically creates HTML buttons for answers using JavaScript.
  • Shows the loading messages and error messages if something goes wrong.
  • Also keeps track of the user's score and displays a final result.

What happens if something goes wrong

The app includes simple error handling:

  • Like if categories cannot be loaded, an error message will be displayed to you.
  • If no questions are returned, the user is told to try again with different settings.
  • If the API request fails (for example, no stable connection), It asks you to refresh the page and try again.
